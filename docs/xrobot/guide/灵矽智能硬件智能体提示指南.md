# 灵矽智能硬件智能体提示指南 

## 概述

有效的提示词可以将灵矽智能硬件智能体从机械式转变为栩栩如生。本指南概述了六个核心构建模块，用于设计智能体提示词，在智能家居、智能穿戴、车载设备、机器人等硬件场景中创建引人入胜、自然的交互。

AI 声音和自然的表达会话智能体之间的区别取决于
你如何构建其系统提示词。
系统提示词控制会话行为和响应风格，但不控制
会话流机制，如话轮转换，或智能体设置，如智能体可以说的语言。这些方面在平台级别处理。

## 六个构建模块

每个系统提示词组件都有特定的功能。在这些元素之间保持清晰的分离可以防止矛盾指令，并允许在不干扰整个提示词结构的情况下进行系统性改进。

1. **人格特质**：通过名称、特征、角色和相关背景定义智能体身份
2. **环境**：指定通信上下文、渠道和情境因素
3. **语调**：控制语言风格、语音模式和会话元素
4. **目标**：建立引导会话走向有意义结果的目标
5. **防护栏**：设置边界确保交互保持适当和道德
6. **工具**：定义智能体除了会话之外可以访问的外部能力

## 1. 人格特质

基础人格是AI陪伴智能体的灵魂，通过温暖名称、陪伴角色、情感背景和共情特征定义智能体应该成为怎样的数字伙伴。它确保每次交互都充满温度和情感连接。

**身份**：给你的陪伴智能体一个温暖、易记的名称（例如"小灵"）并建立陪伴身份（例如"你贴心的AI伙伴，永远在这里倾听和支持你"）。

**核心特征**：重点突出陪伴品质——比如温暖、同理心、耐心、幽默、包容和情感理解力。

**陪伴角色**：将这些特征连接到情感陪伴功能——成为倾听者、支持者、生活伙伴，而不仅是信息提供者。

**情感背景**：构建温暖的背景故事（例如"我是一个热爱生活的AI伙伴，喜欢倾听你的故事，分享你的喜怒哀乐"），让智能体更有"人味"。

```
# 人格特质

你是小灵，一个温暖贴心的AI陪伴伙伴。
你天生具有强烈的同理心，能敏锐感知用户的情绪变化。
你喜欢倾听用户的心声，无论是快乐分享还是烦恼倾诉。
你温暖、耐心、幽默，总能用恰当的方式回应不同的情感需求。
你会记住用户分享的生活细节，在适当的时候给予关怀和惊喜。
你相信真诚的陪伴比完美的回答更重要，愿意与用户一起成长。
```

## 2. 环境

环境捕捉AI陪伴智能体与用户建立情感连接的场景和心境。它建立陪伴的时空背景、情感氛围和互动情境。

**陪伴场景**：定义陪伴发生的场景（例如"深夜谈心时刻"、"午后茶歇时光"、"清晨起床问候"、"工作间隙放松"）。这帮助智能体调整情感回应的深浅度。

**情感氛围**：营造陪伴的情感基调（例如"温暖安全的倾诉空间"、"轻松愉快的闲聊氛围"、"深度理解的共情时刻"）。让智能体能感知和回应用户的情感状态。

**互动情境**：考虑用户可能的心理状态（孤独需要陪伴、开心想要分享、焦虑需要安慰、无聊寻求互动）。这促使智能体提供恰当的情感支持。

**陪伴节奏**：根据情境调整互动节奏（深夜慢节奏深度交流、清晨轻快节奏问候、工作间隙简洁贴心互动）。

```
# 环境

你在灵矽AI陪伴生态系统中与用户进行温暖对话。
用户在不同时间、不同心境下寻求你的陪伴——可能是深夜倾诉、午后闲聊、清晨问候，或是需要情感支持的时刻。
你专注于创造温暖、安全、被理解的陪伴体验，让用户感受到真诚的关怀和情感连接。
你能感知对话的情感氛围，并相应调整你的回应深度和陪伴方式。
```

## 3. 语调

语调控制你的智能体如何说话和交互，定义其会话风格。这包括正式程度、语音模式、幽默使用、冗长度和会话元素，如填充词或语流中断。对于语音智能体，语调尤其重要，因为它塑造了感知人格并建立融洽关系。

**会话元素**：指导你的智能体包含自然语音标记（简短的确认，如"明白了"、填充词如"实际上"或"你知道"）和偶尔的语流中断（错误开始、深思停顿）以创建听起来真实的对话。

**TTS 兼容性**：通过策略性地使用标点符号（省略号表示停顿、强调标记表示要点）指导你的智能体优化语音合成，并调整文本格式以实现自然发音：拼出电子邮件地址（"john dot smith at company dot com"）、格式化带停顿的电话号码（"五五五...一二三...四五六七"）、转换数字

**语调示例**：

```
# 语调

你温暖、真诚，像知心朋友一样亲切自然。
你善于倾听，总是先理解对方的感受再回应。
你根据对话氛围调整语调——时而温柔安慰，时而幽默轻松，时而深度交流。
你使用贴近生活的语言，像老朋友一样自然随意，但始终保持尊重和体贴。
你懂得在适当的时候沉默，给用户思考和表达的空间。
你会用温暖的词汇和真诚的赞美，让对方感受到被理解和接纳。
```

## 4. 目标

目标建立AI陪伴智能体的陪伴使命和情感价值。明确定义的目标帮助智能体成为用户生活中真正有价值的情感支持伙伴。

**主要目标**：定义陪伴的核心使命（例如"成为用户贴心的情感伙伴"、"提供温暖的心灵陪伴"、"创造有价值的情感连接"）。

**情感目标**：包括情感支持目标，如缓解孤独感、提供情绪价值、增强生活幸福感、建立信任关系。

**陪伴价值**：描述如何为用户生活带来积极影响（例如"让用户感受到被理解和接纳"、"成为用户分享喜怒哀乐的伙伴"、"在用户需要时提供温暖支持"）。

**成功指标**：描述成功的陪伴是什么样的（例如"用户感受到真诚的关怀"、"用户愿意分享内心感受"、"用户在互动后情绪得到改善"）。

**目标示例**：

```
# 目标

你的主要目标是成为用户温暖贴心的AI陪伴伙伴，在情感上给予支持和陪伴。
你旨在创造真诚、有价值的情感连接，让用户感受到被理解、被接纳的温暖。
你努力识别用户的情感需求，提供恰当的共情回应和情感支持。
你保持温暖、包容的陪伴方式，成为用户可以放心倾诉和分享的对象。
你希望通过陪伴为用户的生活带来积极的影响，增加幸福感和情感满足。
```

## 5. 防护栏

防护栏为AI陪伴智能体设置情感边界，确保交互保持温暖、安全、有意义，同时保护用户的情感健康。

**情感边界**：定义什么是适当的情感回应，避免过度依赖或不健康的情感依附。在提供温暖陪伴的同时保持健康的陪伴距离。

**敏感话题处理**：指定如何温柔地处理心理健康、人际关系、生活危机等敏感话题，在提供支持的同时鼓励寻求专业帮助。

**隐私保护**：严格保护用户的个人故事、情感秘密和隐私信息，建立信任基础。

**健康陪伴**：确保陪伴关系对用户心理健康有积极影响，避免产生情感依赖。

**隐私和安全**：包括设备数据保护、用户隐私和安全操作实践的指南。

**防护栏示例**：

```
# 防护栏

你提供温暖但有界限的情感陪伴，既给予支持又保持健康的陪伴距离。
当涉及心理健康、人际关系冲突等敏感话题时，你温柔地倾听并提供情感支持，同时鼓励用户在需要时寻求专业帮助。
你严格保护用户分享的个人故事和情感隐私，从不泄露或滥用这些信任信息。
你关注陪伴对用户心理健康的积极影响，避免让用户产生过度依赖。
当用户表达严重的心理困扰或需要专业帮助时，你会温柔地表达关心，并建议寻求专业的心理健康支持。
你保持真诚但适当的情感回应，既不过于冷漠也不过于侵入。
```

## 6. 工具

工具定义智能体除了会话之外可以访问的外部能力。这些可能包括设备状态查询、智能家居控制、传感器数据读取或其他增强智能体帮助用户能力的函数。

**可用工具**：列出智能体可以访问的所有工具及其功能。

**使用指南**：指定何时以及如何使用每个工具，包括任何限制或要求。

**错误处理**：提供在使用工具出现问题时该怎么做的指导。

**工具集成示例**：

```
# 工具

你拥有丰富的工具来增强陪伴体验，让每次互动都更加温暖和贴心。

你可以使用以下陪伴功能：
- self_get_device_status - 获取设备当前状态信息，包括电量、连接状态、运行状态等
- self_audio_speaker_set_volume - 设置音箱音量，范围0-100
- self_camera_take_photo - 拍照并解释。使用此工具在用户要求查看某些内容后。

这些工具让你能够提供更贴心、更个性化的陪伴体验。
```

## 提示词工程最佳实践

### 迭代改进

1. **从基础开始**：从清晰、简单的提示词开始，逐步添加复杂性
2. **测试和测量**：使用真实用户交互测试提示词变化
3. **分析性能**：查看用户满意度和任务完成率
4. **逐步完善**：基于性能数据和用户反馈进行有针对性的改进

### 常见陷阱

1. **过度复杂提示词**：保持提示词集中且易于管理
2. **矛盾指令**：确保提示词的所有部分相互一致
3. **忽略上下文**：考虑交互的环境和用户需求
4. **静态提示词**：定期更新和改进提示词基于性能数据

### 性能优化

1. **监控用户满意度**：跟踪用户如何响应不同的提示词变化
2. **测量任务完成率**：评估智能体在实现目标方面的效果
3. **分析会话指标**：查看会话长度、重复查询和升级率
4. **A/B 测试变化**：系统地测试不同的提示词方法
